## 요청 내용

Windows 11 스마트 앱 컨트롤(SAC)이 gdc-mcp MCP 서버 기동을 차단하는 문제를 보안 수준 저하 없이 해결한다.

## 배경

- CodeIntegrity 이벤트 로그(3077/3033)로 차단 대상 확인: uv/Python이 아니라 `.venv\Scripts\gdc-mcp.exe` (uv가 플러그인 설치 시 로컬 생성하는 서명 없는 entry point shim).
- 로컬 생성 파일이라 서명·평판 확보가 불가능해 SAC가 무조건 차단.
- 같은 venv의 `python.exe`는 Python Software Foundation 서명이 유효해 차단되지 않음.
- 대안 검토: SAC 앱별 예외 등록(기능 없음, 불가), SAC 끄기(재설치 전 재활성화 불가·보안 저하로 비권장), shim 코드 서명(설치 시마다 로컬 생성이라 비현실적).
- 결론: shim을 거치지 않고 서명된 `python.exe`로 `python -m gdc_mcp.server` 직접 실행. `server.py`에 `main()`·`__main__` 가드가 이미 있어 코드 수정 불필요. 사전 기동 테스트로 SAC 차단 없이 정상 동작 확인.
